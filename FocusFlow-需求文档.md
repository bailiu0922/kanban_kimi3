# 项目需求文档：FocusFlow —— 番茄钟 + 任务看板效率工具

> 本文档用于评估 Kimi3 的前端编码能力。请从零开始创建该项目，并严格按以下需求实现。

---

## 一、项目概述

**FocusFlow** 是一个面向个人效率管理的纯前端单页应用（SPA）。它把"任务看板"和"番茄工作法"结合起来：用户把任务放在看板上管理，用番茄钟专注执行，并查看每日专注统计数据。

- **目标用户**：需要自我时间管理的学生 / 自由职业者 / 开发者
- **运行环境**：现代浏览器（Chrome / Edge / Firefox 最新版），无需后端
- **数据持久化**：使用 `localStorage`，刷新不丢数据
- **代码要求**：从零搭建，不依赖现成脚手架生成的最终代码（可用 vite 等初始化，但功能需手写）

---

## 二、技术栈要求

- **方案 B**：Vue 3 + TypeScript + Vite

统一要求：

- 使用 TypeScript
- 状态管理使用框架原生方案（Vue 用 Pinia），**禁止**把所有状态堆在顶层组件
- 样式方案（scss），但需有统一的视觉规范
- 图表库：可用轻量库（如 `echarts` `）
- 拖拽：优先使用原生 HTML5 Drag and Drop API；如需库（如 `@dnd-kit`）需说明

## 三、功能需求

### 3.1 任务看板（Kanban）

看板包含三列：**待办（Todo）/ 进行中（Doing）/ 已完成（Done）**。

| 功能     | 说明                                                                |
| -------- | ------------------------------------------------------------------- |
| 新建任务 | 输入标题（必填，≤50字）、可选描述、可选预估番茄数（正整数，默认 1） |
| 编辑任务 | 点击任务卡可编辑标题/描述/预估番茄数                                |
| 删除任务 | 二次确认后删除（删除 Done 列任务需确认）                            |
| 拖拽移动 | 任务卡可在三列之间拖拽，状态随落点更新                              |
| 排序     | 同一列内可上下拖拽重排（可选增强项）                                |
| 任务计数 | 每列顶部显示该列任务数量                                            |

任务卡片展示：标题、描述（省略号截断）、预估番茄数、已完成番茄数（如 `2/4 🍅`）。

### 3.2 番茄钟（Pomodoro Timer）

| 功能     | 说明                                                                     |
| -------- | ------------------------------------------------------------------------ |
| 专注计时 | 默认 25 分钟，可配置（5–60 分钟）                                        |
| 短休息   | 默认 5 分钟；长休息默认 15 分钟（每完成 4 个番茄触发一次长休息）         |
| 控制     | 开始 / 暂停 / 重置                                                       |
| 关联任务 | 计时开始时需选定一个"当前任务"（从 Doing 列可选，或自由文本）            |
| 番茄完成 | 一个专注周期结束，自动 +1 到当前任务的"已完成番茄数"，并记录一条专注日志 |
| 阶段提示 | 阶段切换时浏览器标题变更（如 `🍅 专注中 12:03`）并可选播放提示音         |
| 跳过休息 | 用户可跳过当前休息直接进入下一轮                                         |

计时器 UI：大号圆形进度环（SVG 实现）、剩余时间、当前阶段标签、当前任务名。

### 3.3 专注统计（Statistics）

- **今日概览**：今日完成番茄总数、总专注分钟数、完成任务数
- **近 7 日柱状图**：每天完成番茄数的柱状图（横轴日期，纵轴数量）
- **热力图（可选增强）**：类似 GitHub 贡献图，按日期深浅展示专注强度
- **任务完成分布**：已完成任务按分类/状态的小计

数据来源：番茄钟完成的专注日志（每完成一个番茄写一条 `{date, taskId, minutes, type}`）。

### 3.4 设置（Settings）

- 专注时长、短休息、长休息可配置并持久化
- 自动开始下一个番茄开关
- 提示音开关
- 数据导出 / 清空（`localStorage` 导出为 JSON，可清空重置）

## 四、UI / UX 规范

- **整体风格**：简洁清爽，浅色为主，主色建议蓝/绿系；支持深色模式切换（增强项）
- **响应式**：桌面端三列并排；移动端（≤768px）列改为纵向堆叠或 tab 切换
- **空状态**：看板为空、统计无数据时显示友好引导文案
- **加载/过渡**：任务增删、计时阶段切换应有基础动画过渡
- **可访问性**：按钮有明确 `aria-label`，计时器进度对屏幕阅读器可读（基础即可）
